# Output Format Templates

Each module is a **standalone deliverable** the user can request. Never generate all at once unless explicitly asked.

---

| ... | ... | ... | ... | ... |

### PPT 明确提示

## 2. 🗺 章节导航图 (Chapter Map)

```markdown
## 🗺 章节导航图

```
📂 课程名称 (N页, M考点)
├── 📁 第1章 [名称] (P1-P10, 8考点)
│   ├── 📄 1.1 [节名] (P2-P5)
│   │   ├── 🔴 核心考点: [名称] (P3)
│   │   ├── 🟡 重要概念: [名称] (P4)
│   │   └── ⚪ 背景了解: [名称] (P5)
│   ├── 📄 1.2 [节名] (P6-P8)
│   │   └── 🔴 核心考点: [名称] (P7-P8)
│   └── 📄 1.3 [节名] (P9-P10)
├── 📁 第2章 [名称] (P11-P28, 15考点)
│   └── ...
└── 📁 第N章 [名称] (P...)
```

### 优先级说明
- 🔴 **P0 核心考点** — 优先复习，极可能出现在试卷上
- 🟡 **P1 重要概念** — 需要理解掌握，常结合 P0 考
- ⚪ **P2 背景了解** — 有印象即可，不会直接出题
```

## 7. 🃏 Anki 卡片导出 (Anki TSV)

Tab-separated, one card per line. Anki import format: `deck name; front; back; tags`

```tsv
"课程名::第1章","Q: [知识点名称]的定义是什么？","A: [定义原文] 来源：P15","ppt-review"
"课程名::第1章","Q: [知识点]和[知识点]的区别","A: [对比表格浓缩]","ppt-review confuse"
"课程名::第2章","Q: [公式]的适用条件？","A: [适用条件说明]","ppt-review formula"
```

Generate 3 card types per P0 point:
1. **Definition**: "What is X?" → definition
2. **Application**: "When to use X?" → scenario + condition
3. **Comparison**: "X vs Y" → key distinction

Output as code block for easy copy-paste, or write to file if user provides path.

## Format Selection Heuristic

| If user says... | Recommend... |
|----------------|-------------|
| "帮我整理重点" | #1 + #3 |
| "我只有1小时了" | #6 (一页纸) |
| "要做Anki卡" | #7 |
| "想刷题" | #8 |
| "对比容易混的" | #5 |
| "公式太多记不住" | #4 |
| "想先看整体结构" | #2 |
| "完整复习" | #1 + #2 + #3 + #4 |

When unsure, default to: **#1 Exam Intel + #3 Core Notes** (highest value per token).
